## Что изменилось

### Проблема
При загрузке PDF с 91 страницей приложение конвертировало **все страницы сразу**:
- ⏱️ Время загрузки: 3-5 минут
- 💾 Память: ~500MB
- ❌ Возможен таймаут

### Решение: Ленивая загрузка страниц

Теперь страницы конвертируются **по требованию**:
- ⚡ Время загрузки: 2-3 секунды
- 💾 Память: ~5-10MB (только текущая страница)
- ✅ Работает с PDF любого размера

## Технические изменения

### 1. Session Data
```python
# Добавлено:
'pdf_path': None,          # Путь к оригинальному PDF
'pdf_page_count': 0,       # Количество страниц
'page_cache': {}           # Кэш конвертированных страниц

# Удалено:
'pdf_pages': []            # Больше не храним все страницы
```

### 2. Upload функция
```python
# Теперь конвертируется только первая страница:
page_count = pdf_processor.get_pdf_page_count(filepath)
first_page = pdf_processor.pdf_page_to_image(filepath, 1, ...)

# Кэшируем первую страницу:
session_data['page_cache'] = {0: first_page}
```

### 3. Change Page функция
```python
# Проверяем кэш:
if page_number in session_data['page_cache']:
    image_path = session_data['page_cache'][page_number]
else:
    # Конвертируем только при необходимости:
    image_path = pdf_processor.pdf_page_to_image(...)
    session_data['page_cache'][page_number] = image_path
```

## Как это работает

### Загрузка PDF
1. Пользователь загружает PDF
2. Определяется количество страниц (быстро)
3. Конвертируется только первая страница
4. PDF открывается мгновенно

### Навигация по страницам
1. Пользователь нажимает "Вперед"
2. Проверяется кэш:
   - Если страница уже конвертирована → мгновенный показ
   - Если нет → конвертация (2-3 сек) → показ
3. Страница добавляется в кэш

### Кэширование
- Просмотренные страницы остаются в памяти
- Повторное открытие страницы мгновенное
- Кэш очищается при загрузке нового файла
